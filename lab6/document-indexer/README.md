# Document Indexer

Сервис, отвечающий за индексацию документов. Принимает запросы асинхронно через AMQP.

## Запуск в Docker контейнере

Для запуска сервиса в Docker контейнере необходимо его для начала собрать
через следующую команду

```bash
docker build . -t document-indexer:dev
```

### Конфигурация контейнера

Собранный Docker-образ поддерживает следующую настройку параметров для контейнеров:

- `RMQ_CONNECTION_URL` - URL для подключения к RabbitMQ в формате `amqp://{user}:{password}@{host}:port`.
- `MEILISEARCH_URL` - URL для подключения к Meilisearch в формате `http://{host}:{port}`.
- `MEILISEARCH_API_KEY` - API ключ для подключения к Meilisearch.

Также, сервис поддерживает указание параметров через конфигурационный файл следующего формата

```yaml
pool_name: <pool-name>
document_schema:
  foo:
    type: text
  bar:
    type: integer
    filterable: true
index_name: <index-name>
```

#### Параметры конфигурационного файла

- `pool_name` - название пула инстансов сервисов. Пул сервисов использует одну и ту же очередь
  для обработки запросов.

- `document_schema` - схема документов, которые индексируются. Представляет собой словарь, где
  ключ обозначает название поля, а значение - информация о поле.

  - `document_schema.<key>.type` - обозначает тип значения поля. Возможные значения: `text` для
    текстовых данных, и `integer` - для числовых.

  - `document_schema.<key>.filterable` - поддерживается ли фильтрация по данному полю. Поддерживается
    только для полей типа `integer`.

- `index_name` - название индекса, в котором будут находиться документы.
